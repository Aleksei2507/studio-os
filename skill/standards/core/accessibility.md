# Accessibility Standard

## Goal

Make accepted user outcomes available across supported input methods, devices, and user needs.

## Rules

- Use semantic platform controls and expose accessible names, roles, values, and state.
- Support keyboard or equivalent non-pointer interaction where the platform provides it.
- Preserve visible focus, logical navigation order, and predictable focus movement.
- Do not use color, position, motion, or sound as the only carrier of meaning.
- Maintain readable contrast, scalable text, and layouts that tolerate zoom or system text sizing.
- Provide clear validation, error identification, and recovery.
- Respect reduced-motion and other relevant system preferences.
- Make dynamic updates perceivable without causing disruptive announcements.
- Test critical flows with platform accessibility tooling and at least one non-default interaction method when applicable.

## Evidence

- accessibility constraints in Design Strategy;
- detailed accessibility behavior in Interface Design when selected;
- implementation semantics and interaction behavior;
- QA observations for critical flows;
- documented exceptions and residual risk.
