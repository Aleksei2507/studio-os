# Research Patterns

## Decision-First Pattern

```md
Decision: Should the first release target independent consultants or small agencies?

Evidence needed:
- observable differences in workflow;
- current alternatives;
- buying and switching constraints;
- competitor concentration.
```

Research only what can change this decision.

## Competitor Pattern

Compare:

- target segment;
- problem addressed;
- primary workflow;
- pricing model when verified;
- strengths relevant to the user problem;
- unresolved gaps;
- switching barriers;
- source and verification date.

Include direct competitors, indirect alternatives, and current manual behavior.

## Niche Pattern

Define a niche as:

```text
Specific user
+ recurring problem
+ context or constraint
+ reason current alternatives are insufficient
```

Do not define a niche only by demographic labels or a broad market category.
