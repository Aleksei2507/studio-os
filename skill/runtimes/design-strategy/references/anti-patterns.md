# Design Strategy Anti-Patterns

## Style Before Use

Failure: choosing colors, trends, or visual mood before defining user context and primary flow.

Recovery: return to the user's task, environment, trust needs, and product positioning.

## Component Design Leakage

Failure: specifying exact components, spacing, CSS, or screen layouts.

Recovery: record the underlying UX principle and leave detailed design to implementation work.

## User As Art Director

Failure: asking the user what style they want without a recommendation.

Recovery: propose one direction based on product evidence and ask for correction or confirmation.

## Brownfield Reinvention

Failure: replacing existing conventions because another style appears more modern.

Recovery: preserve stable patterns unless the accepted product change requires a different interaction model.
