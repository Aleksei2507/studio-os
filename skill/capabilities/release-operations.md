# Release Operations Capability

## Purpose

Assess readiness and prepare or execute an authorized release safely.

## Procedure

1. Read Validation and QA evidence.
2. Verify required configuration, migrations, compatibility, rollback, and known issues.
3. Prepare versioned release notes and handoff instructions.
4. Require explicit authorization before publishing, deploying, or sending external notifications.
5. Capture the resulting release state.

## Evidence

- READY, CONDITIONAL, or BLOCKED decision.
- Release notes.
- Deployment or handoff checklist.
- Explicit authorization for external release actions.
- Version, environment, and known limitations.
