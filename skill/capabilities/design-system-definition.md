# Design System Definition Capability

## Purpose

Define coherent visual foundations and reusable interface patterns from accepted product, brand, platform, accessibility, and Brownfield evidence.

## Procedure

1. Inspect accepted Design Strategy and existing design-system evidence.
2. Define semantic color, typography, spacing, shape, elevation, motion, icon, and content roles required by scope.
3. Define reusable component behavior and variants without prescribing production code.
4. Trace foundations to accessibility, platform, content-density, and product rationale.
5. Record additions, preserved conventions, and deliberate deviations.

## Evidence

- semantic design foundations;
- component and content patterns;
- rationale tied to accepted inputs;
- preserved Brownfield conventions or accepted deviations;
- implementation and QA constraints.
