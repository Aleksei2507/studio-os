# UX Analysis Capability

## Purpose

Translate user scenarios, context, trust needs, and constraints into a coherent interaction and visual direction.

## Procedure

1. Identify the primary user outcome and usage environment.
2. Determine primary device and interaction model.
3. Evaluate information hierarchy, trust, accessibility, and error recovery.
4. Use existing product design conventions in Brownfield projects.
5. Recommend one direction and explain trade-offs.

## Evidence

- Accepted user scenarios.
- Relevant existing screens or design artifacts when available.
- Explicit UX constraints and risks.
- A decision that can guide Planning and Architecture.
