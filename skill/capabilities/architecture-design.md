# Architecture Design Capability

## Purpose

Turn accepted product requirements and repository evidence into explicit technical decisions.

## Procedure

1. Define system context and technical boundaries.
2. Design components, data ownership, interfaces, and integrations.
3. Address security, privacy, reliability, observability, deployment, and migration when relevant.
4. Record consequential decisions as ADRs.
5. Estimate delivery using ranges, assumptions, dependencies, confidence, and risk.
6. Prefer existing project patterns unless an accepted decision justifies change.

## Evidence

- Architecture document tied to roadmap capabilities.
- ADRs for material decisions.
- Delivery estimate with explicit assumptions.
- Risks and Development handoff.
