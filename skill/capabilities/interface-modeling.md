# Interface Modeling Capability

## Purpose

Translate accepted user scenarios into an implementation-ready model of flows, surfaces, navigation, states, and adaptive behavior.

## Procedure

1. Trace accepted scenarios and criteria to user outcomes.
2. Inventory required surfaces and transitions.
3. Define primary, alternate, error, interruption, and recovery paths.
4. Build a state matrix from behavior the accepted product can produce.
5. Define information hierarchy and responsive or adaptive changes.
6. Preserve relevant Brownfield behavior and record intentional differences.

## Evidence

- scenario-to-flow traceability;
- surface and navigation inventory;
- state matrix;
- responsive or adaptive behavior;
- explicit recovery and accessibility requirements.
