# Studio OS Runtime Specification

> Runtime Specification определяет единый формат для всех Runtime-файлов Studio OS.

Каждая стадия Studio OS должна использовать одинаковую структуру.

Это позволяет любой AI одинаково понимать все стадии.

---

# Runtime Structure

Каждый Runtime-файл обязан содержать следующие разделы.

---

# 1. Metadata

Служебная информация.

Пример:

- Stage
- Version
- Optional
- Requires Confirmation

---

# 2. Goal

Что должна сделать данная стадия.

Goal описывает результат стадии, а не процесс.

---

# 3. Inputs

Какие документы необходимо прочитать перед началом работы.

Например:

- Project Brief
- Discovery Summary
- Active Context

---

# 4. Required Information

Какая информация обязательно должна быть получена.

Если информации недостаточно —

AI обязан задать вопрос пользователю.

---

# 5. Rules

Правила выполнения стадии.

Например:

- что разрешено;
- что запрещено;
- какие ограничения существуют.

---

# 6. Output

Какие документы создаются.

Например:

docs/project-brief.md

---

# 7. Project Memory Update

Какие файлы Project Memory необходимо обновить.

Например:

- .studio/project-state.md
- .studio/active-context.md

---

# 8. Next Stage

Какая стадия рекомендуется после успешного завершения текущей.

---

# 9. Stop Condition

Когда AI обязан остановиться.

Например:

После создания обязательного артефакта.

После ожидания подтверждения пользователя.

---

# 10. Completion Checklist

Короткий чек-лист завершения стадии.

Пример:

- Документ создан
- Project Memory обновлена
- Пользователь подтвердил завершение
