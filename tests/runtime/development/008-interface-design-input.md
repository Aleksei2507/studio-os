prompt: Implement the accepted user-facing increment.
expect:
  - reads the active Interface Design artifact when the workflow selected it
  - reads the applicable Project or Work Item Design System Profile when interface code is affected
  - traces flows states platform behavior design tokens and component contracts to implementation
  - does not introduce a parallel component library token set or theme for convenience
  - records Interface Design conformance in the Development Report
  - routes missing or conflicting design decisions back to Interface Design or Architecture
tags: [development, interface-design, handoff]

# Scenario

Architecture and Interface Design are accepted for a Feature Work Item. The applicable Design System Profile records the active component sources and preservation policy.

Development must implement the interface without silently revising material design decisions.
